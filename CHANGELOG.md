# Changelog

## v1.12.20
- **fix**: `wg syncconf` временный конфиг создаётся в `/etc/wireguard/` (каноническая директория), а не в `/tmp` — устранён `fopen: Permission denied` при чтении `wg` из-за ограничений окружения; права `0600`

## v1.12.19
- **fix**: ресинхронизация пиров на старте — сначала `wg syncconf` (как и было), при его сбое per-peer `wg set` как запасной вариант, чтобы интерфейс не оставался без пиров; лог теперь содержит вывод `wg` при ошибке

## v1.12.18
- **fix**: `wg syncconf` (добавление/удаление пира) больше не падает с `fopen: Permission denied` — временный конфиг теперь создаётся с правами `0644`, чтобы `wg` мог прочитать его под другим UID
- **fix**: на старте запущенный интерфейс `wg0` синхронизируется с `wg0.conf` через `wg syncconf` (без wg-quick-директив Address/PostUp), чтобы все пиры из конфига попали на интерфейс

## v1.12.17
- **fix**: корректный серверный публичный ключ при импорте VPN на роутер Keenetic — `wg show ... public-key` возвращает `(none)`, если интерфейс без ключа; теперь берётся ключ из сохранённого приватного (`data/server_private.key`)
- **fix**: паника на старте (`slice bounds out of range`) при отсутствии ключа на интерфейсе — защищены срезы в `getActualServerPrivateKey`/`syncServerKeyFromRunning`, `(none)` трактуется как пустое значение
- **fix**: на старте повторно применяется приватный ключ к уже запущенному, но «безключевому» интерфейсу `wg0`, чтобы VPN-сервер был работоспособен

## v1.12.16
- **fix**: VPN-тогл активируется сразу после сохранения данных роутера (перерисовка списка пиров), без перезагрузки страницы
- **fix**: авто-обновление перерисовывает таблицу при изменении данных пира (домен/логин/пароль роутера), а не только при изменении набора ID

## v1.12.15
- **fix**: `removePeer` использует `syncconfRemovePeer` (build из `wg showconf`), исключает ошибку парсинга `Address=...`

## v1.12.14
- **fix**: добавление пира переведено на `wg syncconf` (build из `wg showconf`), исключает кратковременный просад трафика

## v1.12.13
- **fix**: `wg syncconf` для удаления пира теперь строится из `wg showconf` (формат `wg show`, без `Address`/`PostUp`/`PostDown`), исключает ошибку парсинга `Address=...`

## v1.12.12
- **fix**: добавление пира снова использует `wg set peer allowed-ips` — `wg syncconf` ненадёжно добавляет новых пиров на некоторых версиях WireGuard

## v1.12.11

## v1.12.10
